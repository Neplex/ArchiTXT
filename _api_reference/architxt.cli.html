async architxt.cli.load_or_cache_corpus(archive_file, *, entities_filter=None, relations_filter=None, entities_mapping=None, relations_mapping=None, parser, language, name=None, resolver=None, cache=True)[source]#

Load the corpus from disk or cache.

Parameters:

  • archive_file (Union[BytesIO, BinaryIO]) – An in-memory file object representing the corpus archive.

  • entities_filter (Optional[set[str]]) – A set of entity types to exclude from the output. If None, no filtering is applied.

  • relations_filter (Optional[set[str]]) – A set of relation types to exclude from the output. If None, no filtering is applied.

  • entities_mapping (Optional[dict[str, str]]) – A dictionary mapping entity names to new values. If None, no mapping is applied.

  • relations_mapping (Optional[dict[str, str]]) – A dictionary mapping relation names to new values. If None, no mapping is applied.

  • parser (Parser) – The NLP parser to use.

  • language (str) – The language to use for parsing.

  • name (Optional[str]) – The corpus name.

  • resolver (Optional[EntityResolver]) – An optional entity resolver to use.

  • cache (bool) – Whether to cache the computed forest or not.

Return type:

Collection[Tree]

Returns:

A list of parsed trees representing the enriched corpus.
